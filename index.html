模块化的CSS规范

Posted on 2018-09-06 | In css |

前言：css一直被外行人看作是最简单的（不能算的上是编程语言）的语言，这点我非常的不认同，在css1.0之前我可能也会和你这样说，因为当时的css确实是非常的简单，但是现在都发布css3了，何况还有大量基于css的预处理器，比如sass可以你在css中使用for循环/变量这样的语法。css3也加入了动画的api，效果在某些情况下比js好的多，但是存在一定的限制，总之，各有各的好处。

我在是从最近的项目开发中才真正的意识到css的重要性，不是你多快的使用css还原设计稿（细节把控不到位），而是后期对css的维护和让公司里的开发人员轻松的阅读和修改。这点至关重要，因为css比js容易的多，分分钟就可以写几十行的代码，我看了一下我上一个项目用的css(包括自动加前缀）的，一共有3000多行，写个上万行代码也是不用多久的，但是阅读起来并没有让我很轻松，尽管我使用了预处理器：sass。

下面主要是参考网易的css规范，加以整理，将成为我之后写css的规范，我个人觉得非常不错，如果大家觉得不错也可以使用这css的规范。

  1. CSS分类方法
  2. CSS命名规则
  3. CSS代码格式
  4. CSS优化方案

CSS分类方法

css文件分类

通常来说，一个小一点的项目一个css文件就足够了，但是对于一个中等项目来说，我们就要对css文件进行分类。

我们可以按照css的用途把css文件分为”公共型样式“，“特殊型样式”和“皮肤型样式”，我在之前的项目中暂时没有使用到皮肤型样式，后期有需求可以会考虑修改。

  1. 公共型样式：包括reset(重置）和统一的组件，元件，背景之类的样式。
  2. 特殊型样式：当有个别页面，只需要对之前的组件，元件做细微改动时，我们就可以单独给这个页面设置样式，加以限制。
  3. 皮肤型样式：主要是把css中所有颜色，背景之类的样式抽离出来，方便之后应对修改皮肤之类的需求。

css内部分类

  1. 重置(reset)和默认(base)：前者为了消除浏览器之间的差异，后者是为了减少后面重复的劳动。
  2. 统一处理：在这个位置对一些通用性较高的布局/模块/元件作处理。
  3. 布局(grid)：前缀.g-, 分割页面的各部分内容，如头部，主体，侧栏等。
  4. 模块（module)：前缀.m-, 通常是一些重用性大的一个整体，如导航/登录/注册/评论等。
  5. 元件(unit)：前缀.u-, 通常是一个不可再细分的个体，如按钮，图标，输入框等。
  6. 功能(function)：前缀.f-, 通常是一些常用的样式，如padding/margin的设置，清除浮动之类的。
  7. 皮肤(skin)：前缀.s-，设置单独的文字颜色，背景色（图）等。
  8. 状态：前缀.z-，抽离hover，active等之后的状态。

CSS命名规则

不使用ID选择器，使用前缀为“-”的类选择器，单个字母+“-”为前缀，“-”也表示扩展分隔符，前面所说的布局（grid)(.g-);模块（module)（.m-); 元件（unit)(.u-);功能(function)（.f-）;皮肤（skin)（.s-）和状态（.z-）。

后代选择器命名不以单个字母加“-”为前缀，但是长度要大于2,如.m-list item{}，避免和出现混乱，一个语义化的标签也可以是后代选择器如:.m-list li{}，尽量避免单个字母类选择器出现，考虑他的命名是否已被使用，同时还可以在其它地方重用，后代选择器能短尽量短.

后代选择器类名简短但不失语议

相同语义可以用字母或数字区分，如m-list/m-list2等，它们都是不同的列表模块。

模块和元件扩展类的命名应该以`方法+“-”+数字或字母，如.m-list-1,.m-list-2等。

CSS代码格式

选择器/属性和值都要是小写，必要的时候可以选择单行写一个选择器的方法，去除多余的空格，便于模块识别等。

省略值为0的单位，还是背景引入中的引号，颜色值使用#fff这样的缩写，不过最这节最重要的还是下面这点。

根据属性的重要性按顺序书写

只要遵循竖向顺序即可，先写定位布局类属性，然后是自身属性，最后是文本及修饰类属性。

显示属性：display, position, float, clear, top

自身属性：width, height, margin, padding, border

文本及修饰类属性： font, text-align, color, background

注释格式

将注释全部写在对象的上一行，注释内容两端加空格，块级注释保持统一的缩进，原则上每个系列的样式都需要有一个注释，言简意赅的表明名称/用途/注意事项等。

选择器顺序

  • 从大到小（以选择器的范围为准）
  • 从低到高（以等级上的高低为准）
  • 从先到后（以结构上的先后为准）
  • 从父到子（从结构上的嵌套为准）

CSS优化方案

  1. 属性值尽量缩写，减少css文件大小，增加可读性和可维护性
  2. 避免使用*这样耗性能的选择器，和filter:alpha(opacity=10);}这样的属性。
  3. 选择器合并，把一些相同的属性进行合并
  4. 背景图片合并，使用雪碧图，图片大小压缩。
  5. 图片类文件按钮模块或元件合并，便于就进维护。
  6. 能用css写的尽量不用js
